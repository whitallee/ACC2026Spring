General

Failure to comply with these instructions usually results in a failing grade for the assignment, often zero.

The assignments for this course are take-home assignments. You may use any reference material that you have available. However, you are instructed not to obtain assistance from anyone in completing your assignments.

On the other hand, you are encouraged to get assistance from your instructor, from ACC tutors, or from others if needed to understand the material in the course eBooks or the supplementary reading material.

Downloading and Submitting Assignments

Assignments are posted in Blackboard. There is a zip file associated with each assignment that you can download from Blackboard. The zip file contains the specifications for the assignment.

The following two sets of instructions describe the complete cycle of downloading, completing, and submitting an assignment as of 02/19/25 The details may change over time as the Blackboard software is updated.

  1. Select LEARNING ACTIVITIES on the main Content page.
  2. Open the relevant CLO folder.
  3. Open the folder for the module of interest (MLO-1.1 for example).
  4. Scroll down the page until you find a link named AsgXX where XX is the assignment number. Click that link.
  5. Click the View Instructions button on the bottom right. This will expose a page showing a zip file on the left side of the page.
  6. Click the ellipses menu to the right of the zip file and download the zip file.
  7. Click the large X in the upper-left corner to dismiss the page. You will come back to it later to submit your solution.
  8. Extract the contents of the zip file into an empty folder and open the specification file named ProjXX.htm in your browser. Again, XX is the assignment number.
  9. Complete your assignment according to the specifications given in the file named ProjXX.htm.

All of the assignments in this course require that you create a zip file containing the solution files for the assignment. Once you have done that and you are ready to submit your assignment, follow the steps below.

  1. Reopen the assignment page as described above.
  2. Click the View instructions button on the bottom right. This will open a page with a Submission heading near the bottom.
  3. Drag and drop your solution zip file into the space below the heading.
  4. Click the Save button on the small Edit File Options window that opens.
  5. Click the Submit button on the bottom right of the assignment page.
  6. Click the Submit button on the small Submit Assignment window that opens.
  7. Click either the Close button or the Download button in the small receipt window that opens.
  8. Click the large X button in the upper-right to return to the main Content page.

Alternatively you can skip the first few steps in the first list above by selecting Shortcuts on the main Content page and then selecting the assignment of interest.

Submission Confirmation Email

Immediately following the submission of each new assignment attempt, you must open your ACC email and send an email to baldwin@austincc.edu with the subject line constructed as shown below. If you fail to send the email, your attempt will be ignored and won't be graded.

The subject line of your email must be constructed as follows:

name, ITSE-2317-WWW, XXXXX, AsgYY, Attempt ZZ submitted

where:

  • name is the name under which you are enrolled in the course, such as Sam Smith.
  • WWW is the section number, such as 001
  • XXXXX is the five-digit number assigned to the course, such as 86074
  • YY is the assignment number, such as Asg03
  • ZZ is the attempt number such as 02

A sample subject line constructed according to this requirement follows:

Sam Smith, ITSE-2317-001, 86074, Asg03, Attempt 02 submitted

Format of Downloadable Zip File

The zip files for the assignments that you will download contain some or all of the following material:

  • An assignment specification document (Proj01.htm for example).
  • A source code file for the driver class for the program (Proj03.java for example).
  • A text file containing skeleton code for the assignment (Proj03Skeleton.txt for example).
  • Any other files that you may need, such as image files, to complete the assignment.

Each assignment requires that you write one program. Extract the contents of the zip file into an empty folder. Then open the ProjXX.htm file in your browser to view the programming specifications for the assignment.

Don't use NetBeans, Eclipse, or other high-level IDEs

NetBeans, Eclipse, and other high-level IDEs are excellent productivity tools for programmers in the workplace who are working on large projects, who know exactly what they are doing, and who have a need to become more productive in their programming efforts. However, to write the simple programs required for this course, you should need nothing more sophisticated than a text editor and the Oracle Java Development Kit (JDK). The use of a high-level IDE to write the programs required for this course is overkill, can lead to problems, and is not recommended.

Special Instructions Regarding the Submission of Assignments

You may submit each assignment an unlimited number of times.

The due dates for all assignments can be found by selecting Gradebook on the navigation bar.

Your name

Your name is required one or more times in the output for every program and the program specifications will indicate where your name must appear. If there is any confusion as to where your name must appear, please discuss this with me at least one week prior to the assignment deadline. (If your name does not appear, I will assume that the material you submitted does not constitute your own work and you will not get credit for the assignment.)

Separate class and interface definitions

Every class definition (including the driver class that I provide) and every interface definition should be contained in a separate source code file with an extension of .java. Give the file the same name as the class or interface that the file defines.

Each such file must include any required import declarations.

None of the files may include a package declaration.

Submit source code files only

You must submit only your Java source code files encapsulated in a zip file. Do not submit rar files. Do not submit class files. If you do submit class files, they will simply be overwritten when I compile and execute your programs. If you submit rar files, your submission will not be accepted.

Your name in the zip file

Include your name and the assignment number in the name of the zip file that you submit to help me avoid getting your zip file mixed up with another student's zip file when I retrieve them from Blackboard. A suitable example file name is shown below but other formats would be acceptable as well. I just need to be able to connect the zip file to you and to the assignment number.

Asg01_baldwin.zip

Code compatibility

The required Java version will be specified in the syllabus for your course for the semester in which you are enrolled.

Testing your programs

Don't rely on an IDE to test your programs. Test your source code for each program from the command line using Oracle's javac.exe and java.exe programs. Confirm that your programs conform to the program specifications before submitting your assignments.

To avoid possible class file collision problems, delete all of the class files produced by one version of your program before testing the next version of your program or before testing your next program in the same folder.

No partial credit

I will score your programs from the Windows command line. (In case you develop under a different OS, make certain that your programs are compatible with Windows.) Each program will be graded either correct or incorrect.

Partial credit will not be given for source code that does not fully conform to these instructions and to the program specifications. However, you are allowed unlimited attempts for each assignment. Your highest score among the scores for each assignment will be used to compute your final grade.

I will provide feedback and will be happy to discuss your attempts with you if your attempts fail to meet the specifications.

After copying your source code files into a folder identified by YourName on my M-drive , I will test your program by executing a batch file containing code in that folder similar to the following:

copy ..\_Solution\Proj01.java

del *.class
javac -cp .; Proj01.java
java -cp .; Proj01

What does this mean?

Briefly, it means that I will copy my version of ProjXX.java into the folder identified by YourName, overwriting your file having the same name if there is one. Next I will delete any old class files that may be in that folder. Then I will attempt to compile and execute your program using the required version of Java.

As of this writing, a fairly good tutorial on the Windows Command Prompt is available here.

Extraneous Material

Do not submit extra files such as readme.txt files and expect me to pay attention to them.

Do not expect me to pay attention to text that you may choose to enter into the Blackboard text fields when you submit your solution zip file.

Do not expect me to pay attention to comments in your source code files.

The required material

All I expect to receive from you when you submit your assignment is the set of required source code files and the email described earlier. (In many cases, there will only be one source code file with a name like ProjXXRunner.java) There is no reason that you will need to communicate any information about your assignment to me when you submit it. Any need to discuss the requirements of the assignment must be satisfied prior to your submission of the assignment.

Encapsulate an electronic copy of the required material in a zip file and submit it as a Blackboard assignment. If you are running Windows, you don't need a special program to create the zip file. The Windows file compression utility is entirely adequate.

The zip file must be one that I can unzip using the Windows de-compression utility with no effort required on my part other than to open the file and extract the material contained therein. There must be no requirement for any sort of special decoding. If there is such a requirement, I will not accept the assignment. For example, do not submit a rar file.

Be careful what you copy

There is nothing wrong with using the code from an existing program as a template for a new program. Professional programmers do that all the time. However, if you do that, you should do it in a thoughtful and judicious way.

When writing code for an assignment, don't simply copy an entire existing program into your new program with the intent of modifying it to meet the assignment specifications, even if the existing program appears to be close to what you need.

Instead, you should first make certain that you really do understand the requirements of the assignment. Don't write or copy any code until you have that understanding.

Then you should examine each statement in the existing program and make certain that you understand the purpose and behavior of each statement. If you don't understand the purpose and behavior of a statement in an existing program, don't even consider copying it into your assignment program.

If you copy code from an existing program into your assignment program, you should copy statements from the existing program into the new program one statement at a time. You should make certain that each statement is both needed and appropriate for the new program before you copy the statement.

A sample program may be useful in helping you to write the new code for an assignment. However, that code is only helpful if you truly understand the code in the sample program and understand the requirements of the assignment. The availability of code in a sample program can be detrimental to your success in writing the code for an assignment if fail to copy and use that code in a thoughtful and judicious way.
