Welcome

Welcome to the Ebook for Prof. Baldwin's online version of ITSE 2321 at Austin Community College. The contents of this Ebook plus various online resources form the textbook for the class.

This course, and this Ebook were designed according to the standards of The Quality Matters Higher Education Rubric Sixth Edition. As such, the course and the Ebook were designed around properly aligned Course Level Objectives (CLOs), Module Level Objectives (MLOs), and Learning Activities (LAs). The alignment among these elements is shown in a course map that is provided in your Blackboard course for ITSE 2321.

The course and the Ebook are structured with two CLOs, about a dozen MLOs, and many LAs.

Learning resources

The two most important learning resources for this course are this Ebook, which you can download from the course menu on the left side of your Blackboard page, and the online Ebook, Introduction to Computer Science using Java, July 2019, by Bradley Kjell, Central Connecticut State University.

You will find many references to the Kjell Ebook in this Ebook, and you should consider them to be as important as the material in the Ebook. In addition to providing good explanations for many Java programming concepts, the Kjell Ebook contains numerous quizzes, reviews, exercises, and flash cards. I strongly recommend that you study that material in addition to this Ebook to help you learn about Object-Oriented Programming using Java.

How to use this Ebook

The most straightforward way to use this Ebook is to open the Table of Contents in file named index.htm in your browser and study the chapters in order from top to bottom. You should do that. Assume, however, that after studying many of the chapters in the Ebook, you realize that you would like to go back and re-study those chapters that contain the word "concatenate". You need a way to identify those chapters without having to do a manual search.

Searching for keywords

There is an easy way to identify those chapters on a Windows system and presumably on other systems as well. Open the folder containing the files that make up the Ebook and enter the word "concatenate" in the search box, typically in the upper right corner of the explorer window. When the search is completed, you should see a list of about sixteen files that contain the word concatenate.

Now assume that you want to identify the first chapter in the Ebook that contains the word concatenate. Compare the names of the files in the list with the names of the links in the Table of Contents and you will see the first chapter that contains the word concatenate is the chapter in the file named Jb0160.htm. Select that file in the Table of Contents and use your browser search capability to search for the word.

Maybe that didn't provide the information that you were looking for. A second comparison will show that the next chapter containing the word concatenate is the chapter in the file named Jb0210.htm. Open that file and you will find an explanation of String concatenation.

The Appendix

The Ebook also contains an Appendix with about fifteen chapters to be used as optional supplemental study material. Each chapter contains several short programs with questions about the programs, answers to the questions, and explanations of the answers. Because the Appendix is intended to be used as optional supplemental study material, no attempt was made to specifically align the material in the Appendix with the MLOs. However, the order of the chapters in the Appendix is generally aligned with the order of the chapters in the MLOs.

Students are encouraged to make use of the Appendix to enhance their learning experience.

How to use the Kjell Ebook

The most straightforward way to use the Kjell Ebook is to open the Ebook and study the material interactively from top to bottom making sure that you take advantage of the interactive Quizzes, Flash Cards, Reviews, and Exercises along the way.

The alphabetical index

The Kjell Ebook has an extensive alphabetical index at the bottom of the main page. For example, you could use that index to find and study a discussion of boolean.

However, while the index is extensive. it is not all inclusive. For example, it doesn't contain the word concatenate or any variation of that word.

An online Google search

Fortunately there is another approach to finding discussions of various keywords and topics with the Kjell Ebook. For example, do a Google search for the following:

site:http://programmedlessons.org/Java9 concatenate

As of 06/07/22, that search will return links to five pages on that site that have to do with concatenation.

If you perform a Google search on the following

site:http://programmedlessons.org/Java9 boolean

you will have access to the same page as when using the alphabetical index alone.

I recommend that while reading this Ebook you keep a browser Google search window open to

site:http://programmedlessons.org/Java9 xxx

Whenever you come upon a word or term that you don't understand, such as "method overloading" for example, you substitute that term for XXX and re-open the browser to that new location. In many cases, that will lead you to a Kjell discussion of the term.

Deprecated elements

In the world of software development, "deprecated" refers to functions or elements that are in the process of being replaced by newer ones. As time goes on, when you compile the sample program in this Ebook, you may find that they use elements that have been deprecated by Oracle. For example, as of 07/27/21, if you are using Java SE-16 and you compile a program that instantiates an object of the Integer class, you will get a compiler warning similar to the following:

Proj09Runner.java:27: warning: [removal] Integer(int) in 
Integer has been deprecated and marked for removal
              new Integer(++cnt);
              ^

Note that this is a warning and not an error. While the program that produced the above warning will compile and run successfully under Java SE-16, it can be expected that it will no longer compile under some later version of Java.

Insofar as this course is concerned, the use of deprecated elements in assignment code is not a problem as long as the program containing the deprecated element will still compile and run under the latest released version of Java SE.

If you encounter a deprecated element warning when compiling the sample programs in this Ebook, it will be worth your while to visit the Java documentation and learn how to avoid the warning. For example, the Ebook chapter titled "Jb0200: Variables" shows how to deal with a deprecated version of the Double class.

If you encounter a deprecated element in a sample program in this Ebook that will no longer compile under the latest released version of Java SE, please let me know so that I can update the sample program in the Ebook.
