Deprecated elements

In the world of software development, "deprecated" refers to functions or elements that are in the process of being replaced by newer ones. As time goes on, when you compile the sample program in this eBook, you may find that they use elements that have been deprecated by Oracle. For example, as of 07/28/21, if you are using Java SE-16 and you compile a program that instantiates an object of the Integer class, you will get a compiler warning similar to the following:

Proj09Runner.java:27: warning: [removal] Integer(int) in 
Integer has been deprecated and marked for removal
              new Integer(++cnt);
              ^

Note that this is a warning and not an error. While the program that produced the above warning will compile and run successfully under Java SE-16, it can be expected that it will no longer compile under some later version of Java.

If you encounter a deprecated element warning when compiling the sample programs in this eBook, it would be worth your while to visit the Java documentation and learn how to avoid the warning. For example, the ITSE-2321 eBook chapter titled "Jb0200: Variables" shows how to deal with a deprecated version of the Double class.

If you encounter a deprecated element in a sample program in this eBook that will no longer compile under the latest released version of Java SE, please let me know so that I can update the sample program in the eBook.
