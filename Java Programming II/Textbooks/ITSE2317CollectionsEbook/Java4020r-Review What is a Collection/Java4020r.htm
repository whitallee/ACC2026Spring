Answer 7

False. Primitive values cannot be stored in a collection without first encapsulating them in an object. However, Sometime after Java version 8, Oracle introduced Autoboxing and Unboxing into the language. While it is still not possible to store primitive values in a Collection object, it is no longer necessary to purposely wrap the primitive values in an object to store them in a collection. Therefore, you can add a primitive value to a Collection object and it will be automatically converted to its corresponding object wrapper type. Note, however, that unless you make proper use of generics, conversion back to the original primitive type is not nearly as smooth when you extract the value from the collection.
