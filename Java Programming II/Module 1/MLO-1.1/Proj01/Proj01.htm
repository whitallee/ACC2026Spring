Hint for Asg01

After an extended discussion regarding Asg01, one student asked "So should I be creating the runA method? B/c i do not see it in your code"

My response to that question follows:

You don't see it in my code because you don't have a copy of my source code for the file named Proj01Runner.java. However, you do have my source code for the file named Proj01.java.

Here is how I would approach this problem.

The first thing that you need to do is to analyze the given code in the file named Proj01.java. You need to make sure that you can explain the behavior of each and every line of code in that file.

For example, the code down through line 29 creates an array object of type int and populates it with six random values. The array object's reference is stored in a variable named vals.

Line 30 simply prints a blank line.

Line 34 instantiates an object of your class named Proj01Runner. It passes a copy of a reference to the array object that is stored is the variable named vals to the constructor for the object.

You might surmise that the constructor needs to save that reference to the array object for later use.

You might also surmise that the constructor needs to create and print your certification statement since it needs to be printed early and must be printed by your code.

Lines 37, 38, and 39 print the contents of the array referred to by the reference in vals. This produces the first row of numbers in the required output,

1 4 3 3 3 1 for example.

Now here is the answer to your question. Line 45 calls a method named runA on an object of the class that you designed. That means that your class named Proj01Runner must define a method named runA.

What isn't obvious is, what must be the behavior of the runA method?

Here is what we know, or can discern about that behavior. First, it must return a reference to an object that can be stored in a reference variable of type Collection named ref. That means it is assignment compatible with type Collection.

Second, by examining the code in lines 48 through 52 and examining the required second line of output text, we can determine that the object whose reference is returned by the method named runA must contain the same data values that were contained in the array that was passed to and saved by your constructor.

You may have a couple of options as to the type of object whose reference is returned by the method named runA. First, it must be an object that is assignment compatible with type Collection. Second, it must be iterable by the code in lines 48 through 52. Third it must contain the values from the array object in the same order as the array object.

Lines 48 through 52 print the content of that object producing the second row of numbers in the output.

Line 57 calls a method named runB on the object that was instantiated in line 34. That means that you must also define a method named runB in your class named Proj01Runner. That method must also return a reference to an object that is assignment compatible with type Collection and is iterable. Also, given the required output, that object must contain the values shown in the third row of numbers.

Here is the part that requires some serious study to determine the behavior of the method named runB.

What is the relationship between the required values in the third row of output and the values in the second row of output? What do you know about the behavior of the different data structures in the Collections Framework that would result in such a relationship?

The remaining code simply prints the third line of output. It also computes and prints the sum of those values.

RECOMMENDED SCREENSHOT PROCEDURE

The zip file from which you extracted this document contains a Java image-viewer program named ImageViewer.java. That program can be used to display a png file or a jpg file in its natural size.

Copy ImageViewer.java into the folder containing the given file named RequiredOutput01.png. Compile and execute the Java file. Select the png file in the GUI that appears. That will cause the png file to be displayed as a movable, resizable image on the screen.

Arrange that image and your program output in a side-by-side layout, similar to that shown in Figure 3, with the image of the given file on the left and your program output on the right. Insofar as possible, arrange them so that the text on the left aligns with the text on the right. Then take the screenshot to produce the required screenshot file.

If you are running Windows, you can use the Windows Snip & Sketch application to neatly crop the desired portion of the screen and cause it to be written into an output png file. Rename the png file as indicated under THE DELIVERABLES and add it to the zip file that you will submit as a Blackboard assignment.

If you are using a different operating system, you will need to use the screenshot capture facility provided by that operating system. I'm told that macOS has a built-in tool that is quite similar to Windows Snip & Sketch. It's called the Screenshot app, and it offers a variety of features for capturing screenshots. I'm also told that Linux systems have several screenshot tools that offer similar functionalities to the Windows Snip & Sketch application.
